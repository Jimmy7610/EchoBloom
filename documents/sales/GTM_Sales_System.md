# EchoBloom – GTM Sales System

## Dokumentinformation

* Produktnamn: EchoBloom
* Version: 1.0
* Dokumenttyp: Go-To-Market Sales System
* Modell: Founder-led sales
* Fokus: Första 10 betalande kunder
* Primär målgrupp: Tidiga SaaS-bolag

---

# 1. Executive Summary

EchoBlooms första tillväxt ska inte byggas på ads. Den ska byggas på direktkontakt, lärande och snabba iterationer.

Målet är att få de första 10 kunderna genom founder-led outbound sales.

Det innebär:

* identifiera rätt bolag
* kontakta rätt person
* pitcha tydligt problem + ROI
* boka demo snabbt
* få pilotkund
* samla testimonials
* förbättra processen varje vecka

# 8. Messaging Framework

## Formula

Problem + relevance + curiosity + easy CTA.

## LinkedIn Message #1

Hi [Name], noticed you're growing [Company]. Many SaaS teams know where users drop off onboarding, but not why. We built EchoBloom to capture real-time user reactions and turn them into AI insights. Worth showing you a quick demo?

## Follow-Up #1 (3 days later)

Quick follow-up — we've seen teams uncover friction points in onboarding they never saw in analytics. Open to a 15-min look?

## Follow-Up #2

Totally fine if timing is bad. Happy to send a short example report if useful.

# 10. Demo Call System (15–20 min)

## Structure

### 1. Intro (2 min)

What are you currently using to understand onboarding friction?

### 2. Discovery (5 min)

* biggest onboarding challenge?
* where do users drop?
* how do you collect feedback today?

### 3. Demo (7 min)

Show:

* prompt widget
* dashboard insights
* weekly report

### 4. ROI Close (3 min)

If we helped you identify one major friction point this month, would that be valuable?

### 5. CTA (3 min)

Pilot start next week?

# 15. Objection Handling

## "We already use analytics"

Great — analytics shows where users leave. We help show why.

## "We use surveys"

Nice. We focus on in-the-moment prompts during onboarding, not delayed surveys.

## "No budget"

Understood. If reducing churn by even one customer pays for itself, would it be worth testing?

## "Too busy now"

Makes sense. Want a short example report so we can revisit later?
